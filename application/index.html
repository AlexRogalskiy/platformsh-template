Welcome in the redirection.io example platform.sh integration example

This website has been installed using a platform.sh deployment template for the redirection.io agent.

redirection.io is a complete suite for optimizing your website traffic, user experience and SEO efficiency. It offers many useful features for professional websites:

  • log and analyze all the HTTP traffic;
  • Setup HTTP redirections;
  • create advanced SEO rules (meta tags override, schema.org structured data injection, custom HTML code modification, HTTP response headers management, etc.);
  • etc.

Final setup steps

Now this example project is running, here are the next steps:

  • create an account and a project on redirection.io
  • on the "instances" screen of the redirection.io management interface, click on the "Setup on your infrastructure" button and copy the "Project key";
  • configure this key in your newly created platform.sh project by creating a env:REDIRECTIONIO_KEY project variable:

Once the project is re-deployed, send a few HTTP requests to your website, and you should see logs in the redirection.io manager.

Create your first redirection rules

Our documentation provides many explanations on how to begin your journey with redirection.io. We advise you to read:

  • Create a rule: This page explains how to create redirect rules using redirection.io, and how to configure general settings of your redirection plan: URLs case sensitivity, marketing query parameters support and forwarding, publication history, etc.
  • Create your first redirections: Learn how to create your first redirections with redirection.io, and how to build an efficient redirects plan.

If you just want to test, you can import redirect rules by using this file: redirection-io-example-rules.csv.

Once the import is completed and the redirectionio-agent has processed the update, you can test these rules by clicking the following links:

  • Example 301 redirection: this link should produce a 301 redirect to /
  • Example 302 redirection: this link should produce a 302 redirect to /
  • Example 404 redirection: this link should produce a 404 response. This is useful in case you want to send a 404 back to the user for this request without hitting your application backend
  • Example 410 redirection: this link should produce a 410 response. This is useful in case you want to send a 410 back to the user for this request without hitting your application backend.

Learn more about redirection status codes.

Customize your application

This example comes with a very simple application (this webpage...). Of course, you will want to configure your own website instead of this page.

Here are some tips:

  • configure the service dependencies in .platform/services.yaml
  • configure your application type and properties in .platform.app.yaml
  • put you own code in the application directory (or in any place of your choice depending on the .platform.app.yaml content)
